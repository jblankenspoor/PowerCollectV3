Update Claude API Key in Supabase

Follow these steps to update the Claude API key in your Supabase Edge Function. The current error "Invalid bearer token" indicates that the API key stored in Supabase is either expired, revoked, or not set correctly.

Error Diagnosis

You're seeing the error: "Invalid bearer token"

This means your Edge Function is correctly deployed and can connect to the Claude API, but the API key being used is rejected by Claude.

Step 1: Get Your Claude API Key

First, make sure you have a valid Claude API key. You can get this from the Anthropic Console:

  1. Log in to Anthropic Console
  2. Navigate to the API Keys section
  3. Check if your existing key is still valid, or create a new one if needed

Your Current API Key

YOUR_CLAUDE_API_KEY_HERE

The error suggests this key might have expired or been revoked. You may need to generate a new one.

Step 2: Open Supabase Dashboard

Go to your Supabase project dashboard:

Open Supabase Dashboard

Step 3: Navigate to Edge Functions

In the left sidebar, click on Edge Functions.

Step 4: Select Your Edge Function

Click on the claude-api-proxy function in the list.

Step 5: Update Environment Variable

  1. Click on the Settings tab for your function
  2. Look for the Environment Variables section
  3. Find the entry for CLAUDE_API_KEY
  4. If it exists, update its value to your valid API key
  5. If it doesn't exist, add a new variable with name CLAUDE_API_KEY and your API key as the value
  6. Click Save to update the environment variables

Step 6: Test the Updated Function

After updating the API key:

  1. Return to your application
  2. Try generating Power FX code again
  3. If you still encounter issues, use the test page to diagnose further
Open Test Page

Alternative: Update API Key in Client

As a temporary workaround, you could also modify your client-side code to use a direct connection to Claude API, but this is not recommended for production as it would expose your API key in the browser.
